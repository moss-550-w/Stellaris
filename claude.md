## 项目概述

**项目名称：** 游戏级群星运行模拟器  
**产品定位：** 以视觉沉浸与沙盒创造为核心的游戏化太空模拟器。为玩家提供一个物理自洽、视觉震撼、操作自由的“宇宙游乐场”。  
**核心体验：** 一切物理计算服务于肉眼舒适的感知时间和画面表现，让玩家创造美丽而有趣的宇宙，而非运行枯燥的轨道积分器。

---

## 核心设计原则

1. **体验优先：** 所有系统以玩家“看得爽、玩得转”为最高准则。行星运动速率由视觉舒适度决定，而非真实物理时间。  
2. **物理自洽：** 在玩家可观测的时间窗口（游戏内千年）内保持轨道稳定、能量守恒，不追求跨越亿年的绝对精度。  
3. **视觉震撼：** 影视级天体材质，程序化基底叠加艺术控制，杜绝明显程序化痕迹。  
4. **交互自由：** 零门槛创造、实时参数调整、多视角观测，一切编辑即时生效可见。  
5. **性能可控：** 桌面端稳定 60fps，移动端 30fps，多档画质与物理精度。

## 架构设计

采用**五层松耦合架构**，物理与渲染完全解耦：

```
┌─────────────────────────────────────┐
│           UI 数据层                  │  Vue 3 面板、存档、设置
├─────────────────────────────────────┤
│          玩法交互层                  │  视角、编辑、成就、沙盒工具
├─────────────────────────────────────┤
│          渲染表现层 (主线程)          │  Three.js 3D场景、粒子、特效
│  ← 双缓冲+预测插值获取位置 →         │
├─────────────────────────────────────┤
│          物理仿真层 (Web Worker)     │  动力学引擎、碰撞、演化、快照
├─────────────────────────────────────┤
│          基础内核层                  │  数学库、矢量运算、随机生成器
└─────────────────────────────────────┘
```

**关键架构规则：**
- 物理计算与渲染完全解耦，通过结构化消息通信。
- 物理 Worker 维护**双缓冲**（一计算、一传输），渲染线程取最新缓冲后做一阶外推显示，消除通信延迟。
- 撤销/重做基于**完整系统快照**（最多20步），每次快照包含所有天体状态和随机种子。
- 所有半透明对象按显式排序管线渲染，避免深度错误。

## 核心系统约定

### 时间系统：感知时间
- **基准：** 1倍速 = 地球公转约30秒，完全隐藏真实时间倍率。
- 调速：暂停 → 0.1倍 → 1倍（默认） → 10倍 → 100倍 → 最高1万倍。
- 高速（>1000倍）自动降低物理帧率，保护状态稳定并提示“细节省略”。

### 物理精度（两档）
- **娱乐流畅模式（默认）：** 半隐式欧拉积分，远距合并，支持200+天体。
- **平衡标准模式（进阶）：** Verlet 积分，全N体精确计算，支持80+天体，千年尺度能量漂移<0.01%。此模式为成就和挑战判定基准。

### 碰撞系统（三级判定）
- 低速合并（<0.5倍逃逸速度）
- 中速撕裂（0.5~1.5倍逃逸速度）
- 高速毁灭（>1.5倍逃逸速度）
- 材质影响结果（气态易合并、岩石易碎裂、恒星碰撞触发超新星）

### 视觉渲染约定
- **混合纹理策略：** 岩质行星用程序化噪声基底 + 预制特征遮罩；气态/恒星纯程序化。
- **引力透镜：** 仅正对黑洞（夹角<15°）时全屏光偏折，其他角度用屏幕空间扭曲贴花。
- **透明排序：** 背景星空 → 轨道线 → 范围球 → 大气光晕 → 粒子 → UI光效。
- 基础后处理（bloom/色调映射/暗角）合并为**单通道Shader**。

### 交互与玩法
- 挑战关卡均为精确目标（例如：“使用恰好3个天体，建立双星系统，行星位于宜居带”）。
- 成就判定完全透明，UI实时显示进度和中断原因。
- 撤销/重做以系统快照为单位，回滚完整沙盘状态。

## 开发阶段与节奏

**团队假设：** 1~2名全栈 + 1名技术美术。  
**总周期：** 20周（阶段一 ~ 阶段四）

| 阶段 | 时间 | 核心产出 |
|------|------|----------|
| 一：核心原型验证 | 4周 | 单恒星系统、Verlet积分、感知时间、自由视角、岩质行星 Demo |
| 二：可玩沙盒核心 | 5周 | N体、碰撞/洛希极限、天体编辑器、预设场景、Web Worker 分离、轨道预测、撤销/重做 |
| 三：视觉与玩法打磨 | 6周 | 全天体视觉效果、合并后处理、透明排序、成就/挑战、引力弹弓能量计、性能达标 60fps |
| 四：内容填充与发布 | 5周 | 新手引导、科普卡、UI动画、存档完善、截图工具、移动端适配、V1.0 发布 |

**V2.0 规划（后续）：** 天体演化、航天器与探测器、高精度实验模块、在线分享。

## 给 Claude 的协作指引

- 讨论任何修改时，先检查是否违背**体验优先**和**物理自洽**原则。
- 涉及性能的代码，注意主线程不承载重计算，渲染透明排序不可省略。
- 视觉实现优先考虑混合纹理策略，避免承诺纯程序化。
- 时间相关的逻辑一律映射到感知时间系统。
- 物理精度提升不在 V1.0 范围内，除非明确讨论 V2.0 规划。
